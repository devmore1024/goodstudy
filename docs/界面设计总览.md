## 1. 全局设计原则

### 1.1 语音优先（Voice-First）

小花老师 App 的核心交互模式是 **语音优先**。界面设计需始终围绕这一原则展开：

- **语音是主通道**：所有核心学习流程（测评、答疑、精讲）均以语音对话驱动，屏幕内容作为语音的视觉辅助，而非相反。
- **最少触控操作**：学生在学习过程中应尽量通过语音完成交互，手动操作仅用于拍照、翻页、选择等语音难以完成的场景。
- **声纹识别切换身份**：App 通过声纹识别自动区分家庭成员（学生/家长），无需手动切换账号。当识别到不同声纹时，界面自动切换到对应模式。
- **语音状态常驻可见**：在所有核心页面，语音交互入口（浮动气泡或底部面板）始终可见，确保用户随时可以与小花老师对话。

### 1.2 对话式引导（Conversational Guidance）

界面交互遵循"对话式引导"理念，模拟真实老师与学生面对面交流的体验：

- **主动引导，而非等待操作**：小花老师会根据上下文主动发起对话（如"今天我们来复习分数加减法，准备好了吗？"），而不是展示一个静态页面等待用户点击。
- **分步骤渐进展示**：信息不一次性铺满屏幕，而是跟随对话节奏逐步展现。例如，测评结果先口述总评，再逐项展开详情。
- **情感化反馈**：小花老师的语气和表情会根据场景变化——答对时鼓励、答错时温和纠正、长时间未互动时轻声提醒。
- **容错友好**：对话中允许用户打断、重复提问、跑题，小花老师应能自然应对并引导回正轨。

### 1.3 数字人老师展示规则

小花老师的数字形象是 App 的灵魂元素，其展示方式需遵循以下规则：

| 规则编号 | 规则内容 |
|---------|---------|
| R1 | 数字人形象在 App 内有三种展示形态：**大图居中**、**小图悬浮**、**隐藏** |
| R2 | 大图居中形态用于需要"面对面交流"的场景（如欢迎、测评、精讲），占屏幕上半部分约 40% 区域 |
| R3 | 小图悬浮形态用于学生自主操作场景（如拍照、刷题），以 56x56dp 圆形气泡悬浮于右下角 |
| R4 | 隐藏形态用于纯信息展示页面（如报告详情、设置页），此时语音入口仍保留但数字人不可见 |
| R5 | 数字人形象应始终带有口型动画与语音同步，静默时保持轻微呼吸/眨眼待机动画 |
| R6 | 在任何形态下，点击数字人形象均可展开语音交互浮层（G1） |
| R7 | 数字人表情状态包括：默认微笑、说话中、思考中、鼓励、提醒、惊喜，共 6 种 |

### 1.4 设计规范概要

| 项目 | 规范 |
|------|------|
| 设计尺寸 | 375 x 812 pt（iPhone X 基准），适配刘海屏与直屏 |
| 字体 | 系统默认字体（iOS: SF Pro / Android: Roboto），中文使用系统默认 |
| 主色调 | 品牌绿 #2BBB6E（小花老师主题色）、辅助蓝 #4A90D9（知识/学习）、警示橙 #FF8C42 |
| 圆角 | 卡片 16pt、按钮 12pt、输入框 8pt |
| 间距体系 | 基础单位 4pt，常用间距 8/12/16/24/32pt |
| 动效原则 | 自然流畅，时长 200-400ms，缓动曲线 ease-in-out |

App 底部 Tab 栏根据当前身份模式（学生/家长）展示不同的标签页结构。

| 2 | 每日学习 | 书本图标 | D1-每日学习主页 | 今日学习任务列表，含拍照判题、答疑、精讲入口 |

| 4 | 个人中心 | 人像图标 | F1-个人中心 | 账号信息、学习设置、家庭成员管理入口 |

```
┌─────────────────────────────────────────────────┐
│                                                 │
│              （ 页面内容区域 ）                    │
│                                                 │
```

|---------|---------|------|---------|------|
| 1 | 首页 | 房屋图标 | B2-首页-家长模式 | 家长主入口，展示孩子学习概况与小花老师建议 |

### 2.3 模式切换机制

- **自动切换**：声纹识别到家长声音时，自动切换至家长模式；识别到学生声音时，自动切换至学生模式。切换时 Tab 栏平滑过渡动画（300ms）。
- **手动切换**：在个人中心页面提供手动切换入口，用于声纹未注册或识别失败的场景。
- **切换提示**：模式切换时，顶部出现 Toast 提示"已切换到 XX 模式"（显示 2 秒自动消失）。

## 4. 全局组件说明

### 4.1 状态栏（Status Bar）

| 属性 | 规范 |
|------|------|
| 高度 | 跟随系统（iOS 刘海屏 44pt，直屏 20pt；Android 按系统适配） |
| 背景色 | 默认透明，页面内容区延伸至状态栏下方 |
| 文字颜色 | 浅色背景页面使用深色文字；深色/渐变背景使用白色文字 |
| 安全区域 | 所有可交互元素必须位于状态栏安全区域之下 |

### 4.2 导航栏（Navigation Bar）

| 属性 | 规范 |
|------|------|
| 高度 | 44pt（不含状态栏） |
| 背景 | 默认白色 #FFFFFF，支持透明模式（用于大图居中页面） |
| 标题 | 居中显示，字号 17pt，字重 Semibold |
| 左侧按钮 | 返回箭头，热区 44x44pt |
| 右侧按钮 | 根据页面需要配置，最多 2 个，热区 44x44pt |
| 分割线 | 底部 0.5pt 分割线，颜色 #E5E5E5，滚动时显示 |

### 4.3 Tab 栏（Tab Bar）

| 属性 | 规范 |
|------|------|
| 高度 | 49pt + 底部安全区域（iPhone X 及以上为 34pt） |
| 背景 | 白色 #FFFFFF，顶部 0.5pt 分割线 #E5E5E5 |
| 图标尺寸 | 24x24pt |
| 标签文字 | 字号 10pt，字重 Regular |
| 选中态 | 图标+文字变为主色 #2BBB6E |
| 未选中态 | 图标+文字为灰色 #999999 |
| 切换动效 | 选中图标缩放弹跳动画（scale 1.0→1.15→1.0，200ms） |

### 4.4 数字人悬浮气泡（Floating Bubble）

当数字人处于"小图悬浮"形态时，以悬浮气泡形式展示在页面上。

| 属性 | 规范 |
|------|------|
| 尺寸 | 56 x 56 dp 圆形 |
| 默认位置 | 右下角，距右边缘 16pt，距 Tab 栏顶部 16pt |
| 可拖拽 | 是，可沿屏幕边缘上下拖拽，松手后自动吸附至最近边缘 |
| 阴影 | 0 4pt 12pt rgba(0,0,0,0.15) |
| 边框 | 2pt 白色描边 |
| 内容 | 小花老师头像，带呼吸动画（微微缩放，周期 3 秒） |
| 说话状态 | 气泡外圈出现绿色声波环动画 |
| 消息提示 | 气泡上方弹出对话气泡（最多显示 2 行文字，4 秒后自动收起） |
| 点击行为 | 点击展开语音交互浮层（G1） |
| 长按行为 | 长按 500ms 后进入拖拽模式 |
| 层级 | z-index 最高，始终浮于页面内容之上 |

### 4.5 语音交互浮层（详见 G1）

语音交互浮层是全局可呼出的核心交互组件，通过以下方式触发：

- 点击悬浮气泡
- 点击大图居中形态的数字人
- 系统检测到用户语音唤醒词（"小花老师"）

| 属性 | 规范 |
|------|------|
| 展示方式 | 底部上滑浮层，覆盖页面 70% 高度 |
| 背景 | 半透明毛玻璃遮罩 |
| 内容区 | 上方：数字人半身像；中间：对话文字实时转写；下方：语音输入指示器 |
| 关闭方式 | 下滑手势 / 点击遮罩区域 / 语音说"再见" |
| 动画 | 上滑弹出 300ms ease-out |

### 4.6 通用加载与异常状态

| 状态 | 展示方式 | 说明 |
|------|---------|------|
| 页面加载中 | 骨架屏（Skeleton） | 页面首次加载时展示内容占位骨架 |
| 局部加载中 | 区域内旋转指示器 | 某个卡片/模块数据加载时 |
| AI 思考中 | 数字人"思考"表情 + 文字提示"小花老师正在思考..." | AI 响应等待时 |
| 网络异常 | 异常插图 + "网络开小差了" + 重试按钮 | 详见 G4 |
| 空状态 | 场景化插图 + 引导文案 | 如暂无学习记录、暂无报告等 |
